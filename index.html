Describing Relevant Implications

- Privacy

Securing the database is crucial to protect user information from unauthorized access, data breaches, or cyberattacks. Implementing strong encryption and access controls is essential. The database that collects information from users is restricted from them and does not take them to the data folder. Access to the database should be restricted to authorized personnel only. User rights must be respected, including the right to access, edit, or delete their data. Users should have the means to exercise these rights easily. User data is often collected; such as personal information and browsing history, which is stored in databases. The implications for privacy arise from how this data is collected, stored, and protected. Users should be informed about data collection and give consent where necessary. Passwords should be hashed; it may be necessary to anonymize or pseudonymize user data in the database to protect individual identities. Only necessary data for the website's intended purpose must be collected. Minimizing data collection reduces the potential impact on user privacy.

- Functionality

The functionality of the website must align with the client's requirements and expectations. This involves clear communication and documentation of the client's needs throughout the development process. To ensure usefulness, the website's design and features should be tailored to meet the needs and preferences of its intended users. User experience (UX) testing and feedback can help refine functionality. Comprehensive testing, including functionality testing, is crucial to ensure that the website performs as intended. This involves checking that all features work correctly, data is accurately retrieved from and stored in the database, and user interactions are smooth. Beyond functionality, usability testing assesses how easily users can navigate and interact with the website. It helps identify any user experience issues and can lead to improvements in functionality and user-friendliness. The website's functionality should be designed to scale with user demand. As the user base grows, the database and infrastructure should be able to handle increased traffic without performance issues. A slow website can undermine its usefulness. Optimizing code, database queries, and server performance is crucial to maintaining fast loading times and responsive functionality.

- Sustainability and Future Proofing

The database must be designed with a focus on normalization, ensuring that data is organized efficiently into separate tables for different entities, such as people, schedule items, news items, and comments. This normalization facilitates easier maintenance and expansion. The use of foreign keys to reference shared data like names or other details, and ensuring that corrections or updates need to be made in one place only. Adopting a modular approach to database design, with distinct tables for various entities makes it simpler to add new tables or features without the need to alter existing structures, enhancing future-proofing. Maintenance of comprehensive documentation detailing the database schema, relationships, and the purpose of each table is crucial. This documentation aids in database maintenance and makes it easier for future developers to understand and work with the database. (This can be found on this planning website below.) Version control for the database must be implemented, allowing trackage of changes and revertion to previous versions if necessary. This helps maintain consistency and manage updates over time.

- End user consideration

Data retrieved from the database needs to be presented in a user-friendly and organized manner. User interfaces should prioritize clarity and ease of navigation, catering to users' needs for efficient access to information. The design of the website must be responsive so it adapts to various devices and screen sizes. This is essential for providing a seamless user experience, as modern users expect websites to work well on both desktop and mobile devices. Due to modern users’ expectations, data needs to be formatted in a way that aligns with it. Consistent styling, such as fonts, colours, and layout, helps maintain a professional and user-friendly appearance. Forms for adding, updating, or deleting entries in the database must be intuitive and user-friendly. Form validation to help users avoid input errors may be considered. A mechanism for users to undo their actions, especially when updating or deleting data should be provided. This helps prevent accidental data loss and enhances the user experience. Users expect fast response times, and efficient queries contribute to a smoother user experience Optimisation of database queries ensures quick retrieval of data. Clear error messages and notifications to guide users when they encounter issues, such as failed form submissions or data retrieval errors must be implemented. Effective error handling contributes to a frustration-free experience.

Addressing Relevent Implications

Privacy

I have made sure that there are role-based access controls to ensure that only those who need specific data have access to it. I have added authorisation for members. Authorisation 0 is for the superuser. They have access to add news, update, and delete news items. They also have access to add events on the schedule page, while being able to update and delete the events. Authorisation 1 is for general members. They are able to view all the content uploaded by the superuser and they are also able to comment on the news items if they wish. This way, privacy is respected as only those who need specific data have access to it. I have also made sure that the amount of information taken from members is kept to a minimum so their data is protected. I have only asked for first/ last name and email address. These things are needed for identification purposes, but no other information has been asked or retained. Passwords are hashed, so the data is kept hidden, therefore minimising data collection.

Functionality

The outcome meets a client’s expectations as I have tested this by asking fellow guitar group members to test my site. Through this testing, I have received feedback which has helped me refine functionality. Throughout my four sprints, I have done comprehensive testing of the functionality. Each sprint had a CUD test to ensure that the website performed as intended. I have planned for each sprint by making diagrams and flow charts which allow me to visualise everything to avoid mistakes. These flowcharts show how the member navigates to certain pages. The layout of the site is quite simple and easy to navigate. I have created an sql installer file so that small changes and corrections are easy to implement. The website runs fast due to the optimization of the code.

Sustainability

I have built this database, ensuring that data is organised efficiently into separate tables. I have a members table which contains information about different members who have signed up and or logged in. I have scheduled events, news items, and comments. I have also used foreign keys to reference shared data like member_id. All corrections and updates are done in one place without the need to alter existing structures; and sustainability. If a member changes their name, the change automatically connects to related tables such as schedule and news.

End User Consideration

On my website, there is consistent styling, such as fonts, colours, and layout. This helps maintain a professional and user-friendly appearance. I have error messages throughout the website for things such as incorrect information filled out. If a new member tries to login to the website, the details will not be accepted. A message that says “UserID does not exist. Please sign up first,” shows up. There are a few of these error messages throughout the website. These clear error messages are there to guide members when they encounter issues such as wrong credentials or invalid entry details. This effective error handling allows a smoother experience for the user. Users expect fast response times; queries are optimised to ensure quick retrieval of data.
